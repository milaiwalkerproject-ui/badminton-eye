    1. Import Vision, CoreML, CoreVideo, CoreGraphics, Foundation.

    2. Add file-level constants:
       - `static let defaultModelName = "ShuttlecockDetector"` (the .mlmodel file name without extension)
       - `static let confidenceThreshold: Float = 0.5`

    3. Class declaration: `final class CoreMLShuttleDetector: ShuttleDetecting, @unchecked Sendable`
       - Property `let modelName: String`
       - Private property `private var vnModel: VNCoreMLModel?` (lazy-loaded, cached)
       - Private property `private let lock = NSLock()` (thread safety for lazy init)
       - Init takes optional `modelName: String = CoreMLShuttleDetector.defaultModelName`

    4. Private method `private func loadModelIfNeeded() throws -> VNCoreMLModel`:
       - Guard vnModel is nil, else return cached
       - Lock around loading for thread safety
       - Double-check after lock acquisition
       - Use `Bundle.main.url(forResource: modelName, withExtension: "mlmodelc")` to find compiled model
       - If URL is nil, throw CoreMLDetectorError.modelNotFound(modelName)
       - Create MLModel from URL with `MLModel(contentsOf:)`
       - Create VNCoreMLModel from MLModel
       - Cache in vnModel, return

    5. Method `func detect(in pixelBuffer: CVPixelBuffer) async throws -> [ShuttleObservation]`:
       - Call loadModelIfNeeded() to get VNCoreMLModel
       - Create VNCoreMLRequest with the model
       - Set request.imageCropAndScaleOption = .scaleFill
       - Create VNImageRequestHandler(cvPixelBuffer:options:) with empty options dict
       - Perform request synchronously (Vision handles threading): handler.perform([request])
       - Extract results as [VNRecognizedObjectObservation]
       - Filter observations where observation.confidence >= Self.confidenceThreshold
       - Map each observation to ShuttleObservation:
         - position: CGPoint(x: observation.boundingBox.midX, y: 1.0 - observation.boundingBox.midY)
           (Vision uses bottom-left origin, flip Y to top-left for consistency with image coordinates)
         - confidence: observation.confidence
         - frameIndex: 0 (caller sets correct frame index)
       - Return filtered array

    6. Method `func detect(imageSize: CGSize, frameCount: Int) async throws -> [ShuttleObservation]`:
       - Throw CoreMLDetectorError.simulationNotSupported
       - This is the placeholder-only path; real detector requires real frames

    7. Enum `enum CoreMLDetectorError: LocalizedError`:
       - case modelNotFound(String) -- "CoreML model '\(name)' not found in app bundle"
       - case simulationNotSupported -- "CoreMLShuttleDetector requires real video frames"
       - case inferenceFailed(String) -- wraps underlying Vision errors

    Refactor the analyze method in HawkEyePipeline to support real frame extraction when a real detector is available, while preserving the placeholder path exactly as-is for backward compatibility.

    1. Add import at top: `import CoreMedia` (for CMTime frame extraction)

    2. Add constants at class level:
       - `static let frameSkipInterval = 4` -- process every 4th frame at 240fps = 60 detections/sec
       - `static let maxAnalysisFrames = 150` -- cap total frames analyzed to prevent runaway analysis

    3. Add a private computed property `private var usesRealDetection: Bool`:
       - Return `!(detector is PlaceholderShuttleDetector)`
       - This determines which analysis path to take

    4. Add a new private method for real-frame analysis:
       `private func analyzeWithRealFrames(videoURL: URL, calibration: CalibrationProfile, homography: [[Double]], imageSize: CGSize) async -> [ShuttleObservation]`

       Implementation:
       a. Create AVURLAsset(url: videoURL)
       b. Load video track: `try await asset.loadTracks(withMediaType: .video).first`
       c. Get nominal frame rate from track
       d. Compute effective skip interval: if FPS >= 120, use Self.frameSkipInterval; else use 1 (no skip for low FPS)
       e. Create AVAssetReader with the asset
       f. Configure AVAssetReaderTrackOutput with video track, outputSettings:
          `[kCVPixelBufferPixelFormatTypeKey as String: kCVPixelFormatType_32BGRA]`
       g. Add output to reader, startReading()
       h. Loop extracting sample buffers:
          - var frameIndex = 0, var processedCount = 0
          - while let sampleBuffer = output.copyNextSampleBuffer(), processedCount < Self.maxAnalysisFrames
          - frameIndex += 1
          - if frameIndex % effectiveSkipInterval != 0, continue (skip frame)
          - Guard let pixelBuffer = CMSampleBufferGetImageBuffer(sampleBuffer)
          - Call try await detector.detect(in: pixelBuffer)
          - Update each observation's frameIndex (create new ShuttleObservation with correct frameIndex since detector returns 0)
          - Append to results array
          - processedCount += 1
          - Update progress: 0.2 + 0.5 * (Double(processedCount) / Double(Self.maxAnalysisFrames))
       i. Return all collected observations

    5. Modify the existing `analyze(videoURL:calibration:)` method:
       After Step 2 (video metadata extraction, progress = 0.2), branch:

       If `usesRealDetection`:
       - Remove artificial delays (no Task.sleep for real analysis)
       - Call analyzeWithRealFrames to get observations
       - Map observations to image-space positions: multiply each observation.position by imageSize
         (Vision returns normalized 0-1 coords, need image-space for homography)
         `CGPoint(x: obs.position.x * imageSize.width, y: obs.position.y * imageSize.height)`

       - Keep existing code EXACTLY as-is (Steps 3-4 with artificial delays and detect(imageSize:frameCount:))

    6. CRITICAL: Do NOT change the init signature. The default parameter `PlaceholderShuttleDetector()` stays.
