  <action>
    read_first: BadmintonEye/BadmintonEye/Views/HeadToHeadView.swift (for win/loss computation pattern), BadmintonEye/BadmintonEye/Models/SwiftDataModels.swift (for PersistedMatch properties)

    Create MatchStatsViewModel.swift as an @Observable class that receives [PersistedMatch] and computes all analytics. The ViewModel does NOT own the @Query -- it receives matches and a playerName (the "me" player, defaulting to the most frequent playerAName) from the view.

    Properties to expose:
    - completedMatches: [PersistedMatch] -- filtered isComplete == true, sorted by startedAt descending
    - totalWins: Int -- matches where user is on the winnerSide
    - totalLosses: Int -- completed matches minus wins
    - winRate: Double -- wins / total as percentage (0-100), 0 if no matches
    - currentWinStreak: Int -- count consecutive wins from most recent match backwards, streak breaks on first loss
    - hasEnoughData: Bool -- completedMatches.count >= 3

    For Plan 02 (trend chart + scoring patterns), also expose:
    - func winRateOverLast(_ n: Int) -> [(index: Int, rate: Double)] -- rolling win rate for last N matches, returns array of (match index, cumulative win rate at that point) for charting. If n == 0, use all matches.
    - func perGameAverages() -> [(game: Int, avgScored: Double, avgConceded: Double)] -- for games 1, 2, 3: average points scored and conceded by the user across all completed matches. Only include game 2/3 if data exists.

    Win determination logic (match HeadToHeadView pattern):
    - User is on sideA if playerAName or playerA2Name matches the selected player name
    - Win if user is on sideA and winnerSide == "sideA", or user is on sideB and winnerSide == "sideB"

    For perGameAverages, determine user's side per match, then:
    - Game 1: always exists, use game1ScoreA/game1ScoreB (swap if user is sideB)
    - Game 2: only if game2ScoreA != nil
    - Game 3: only if game3ScoreA != nil

    The "me" player detection: scan all completed matches, find the name appearing most frequently in playerAName across matches. Expose as `detectedPlayerName: String?`. Allow override via a `selectedPlayerName: String?` property.

    func update(matches: [PersistedMatch]) -- call this to recompute when matches change.

    - perGameAverages function exists returning [(game: Int, avgScored: Double, avgConceded: Double)]
    - hasEnoughData computed property exists
    - No import of SwiftData (receives plain array, not @Query)
