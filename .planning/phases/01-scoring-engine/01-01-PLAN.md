    3. Implement doubles rotation in MatchEngine.applyScorePoint (GREEN):
       The rotation order for a doubles game is a fixed array of 4 PlayerPositions set at game start:
       ```
       rotation[0] = initial server (teamA, player at right court)
       rotation[1] = player diagonally opposite initial receiver (teamB, player NOT at right court initially)
       rotation[2] = initial server's partner (teamA, other player)
       rotation[3] = initial receiver (teamB, player at right court initially)
       ```
       When serving side scores: server stays same, swap server's court position within their team.
       When receiving side scores: advance servingPlayerIndex = (servingPlayerIndex + 1) % 4.

       For mixed doubles 2026 rule: the rotation order is set up so that when receiving side gains service, the next player in rotation is the NON-receiver. This is inherent in the standard rotation[0-3] order above.

    PITFALL WARNING (from research): The most common bug is confusing "who serves next" when receiving side wins. The rotation array approach avoids this by making it a simple index increment. Do NOT try to compute the next server dynamically -- use the fixed rotation array.
