Create TrajectoryCalculator.swift with these types and methods:

```
struct CourtPoint: Codable {
    let x: Double  // court-space X (0.0 = left sideline, 1.0 = right sideline)
    let y: Double  // court-space Y (0.0 = near baseline, 1.0 = far baseline)
}

enum LandingResult: String {
    case inBounds   // green
    case outOfBounds // red
    case uncertain   // yellow
}

struct HawkEyeResult {
    let trajectoryPoints: [CourtPoint]  // shuttle path in court coordinates
    let landingPoint: CourtPoint        // predicted landing spot
    let landingResult: LandingResult    // IN / OUT / UNCERTAIN
    let confidence: Double              // 0.0 to 1.0
    let marginFromLine: Double          // distance from nearest line in court-normalized units
}
```

Methods on TrajectoryCalculator (struct, no state):
- `func computeHomography(imageCorners: [CGPoint], imageSize: CGSize) -> [[Double]]` -- computes a 3x3 perspective transform matrix mapping image coordinates to normalized court coordinates (0-1 range). Use a simple 4-point perspective transform: solve the 8-equation system from the 4 corner correspondences. Court corners map to (0,0), (1,0), (1,1), (0,1).
- `func transformPoint(_ point: CGPoint, using homography: [[Double]]) -> CourtPoint` -- applies homography matrix to map an image-space point to court-space.
- `func fitTrajectory(_ points: [CourtPoint]) -> (trajectory: [CourtPoint], landing: CourtPoint)` -- fits a smooth parabolic curve through detected points and extrapolates to the landing position. Use quadratic interpolation for the y-component (gravity-like descent) and linear for x.
- `func determineLanding(_ point: CourtPoint) -> (result: LandingResult, margin: Double)` -- checks if landing point is within court boundaries (singles: x in 0.155...0.845, y in 0.0...1.0; doubles: x in 0.0...1.0). Margin < 0.02 normalized units = uncertain. Return margin as distance from nearest boundary line.
- `func computeConfidence(detectionCount: Int, videoFPS: Double, margin: Double) -> Double` -- confidence based on: number of detected shuttle positions (more = better), video frame rate (higher = better), margin from line (farther = more confident). Formula: base = min(1.0, Double(detectionCount) / 15.0) * 0.4 + min(1.0, videoFPS / 60.0) * 0.2 + min(1.0, margin / 0.1) * 0.4. Clamp to 0.0...1.0.
