  <action>
Create AuthManager.swift as an @Observable singleton managing Apple Sign-In state:

1. Properties: `isSignedIn: Bool` (persisted in UserDefaults), `userName: String?`, `userEmail: String?`, `userIdentifier: String?` (all from Apple ID credential)
2. `signIn()` method: Uses ASAuthorizationAppleIDProvider with ASAuthorizationController to present the system Sign in with Apple sheet. On success, stores credential.user in UserDefaults as "appleUserIdentifier", sets isSignedIn = true, populates userName/userEmail from credential.
3. `signOut()` method: Clears UserDefaults "appleUserIdentifier", sets isSignedIn = false, clears user properties.
4. `checkAuthState()` method: On init, reads "appleUserIdentifier" from UserDefaults. If present, calls ASAuthorizationAppleIDProvider().getCredentialState(forUserID:) to verify still authorized. If revoked/not found, auto-signs out.
5. Import AuthenticationServices framework.

Create BadmintonEye.entitlements plist file with:
- com.apple.developer.applesignin = ["Default"]
- com.apple.developer.icloud-services = ["CloudKit"]
- com.apple.developer.icloud-container-identifiers = ["iCloud.com.badmintoneye.app"]

Update project.pbxproj: Add AuthManager.swift to PBXBuildFile and PBXFileReference sections. Add BadmintonEye.entitlements to PBXFileReference. Add CODE_SIGN_ENTITLEMENTS = "BadmintonEye/BadmintonEye.entitlements" to all build configuration sections. Follow the exact pattern used for existing files like WatchSyncManager.swift.

Note: Use AuthenticationServices framework (first-party), not any third-party auth library.
