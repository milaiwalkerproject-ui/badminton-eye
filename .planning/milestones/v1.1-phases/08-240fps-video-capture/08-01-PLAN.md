<objective>
Refactor VideoCaptureManager from file-based recording to delegate-based frame capture with 240fps support, HEVC encoding, and a circular frame buffer.

Purpose: The existing VideoCaptureManager uses AVCaptureMovieFileOutput which cannot access individual frames or control frame rate selection. This plan replaces it with AVCaptureVideoDataOutput + delegate pattern, adds device format enumeration for highest-FPS selection, writes via AVAssetWriter with HEVC, and adds a circular buffer that retains the last 10 seconds of frames.

Output: Refactored VideoCaptureManager.swift and new CircularFrameBuffer.swift
</objective>

  <action>
    read_first: BadmintonEye/BadmintonEye/Services/VideoCaptureManager.swift

    Create a new file `CircularFrameBuffer.swift` in Services/ that implements an in-memory ring buffer of CMSampleBuffers.

    Requirements:
    - Class `CircularFrameBuffer` (not @Observable — internal service, not UI-bound)
    - `@preconcurrency import AVFoundation` for Swift 6 compatibility (established pattern)
    - Init takes `capacity: TimeInterval` defaulting to 10.0 (seconds)
    - `append(_ sampleBuffer: CMSampleBuffer)` — adds buffer to ring. Evicts oldest buffers when total duration exceeds capacity. Use CMSampleBufferGetPresentationTimeStamp to compute duration window.
    - `flush(to outputURL: URL, codec: AVVideoCodecType, width: Int, height: Int, fps: Double) async throws -> URL` — writes all buffered frames to disk via AVAssetWriter with the specified codec. Returns the output URL. After flush, clears the buffer.
    - Internal storage: `[CMSampleBuffer]` array acting as ring. Track oldest timestamp. On append, drop buffers whose PTS is older than (newest PTS - capacity).
    - `var bufferedDuration: TimeInterval` computed property returning current buffer duration.
    - `var isEmpty: Bool` computed property.
    - `func clear()` — drops all buffers.

    The flush method must:
    1. Create AVAssetWriter with outputURL and fileType .mp4
    2. Configure AVAssetWriterInput with video codec, width, height, and expected FPS
    3. Use AVAssetWriterInputPixelBufferAdaptor for efficient pixel buffer appending
    4. Iterate stored buffers, extract pixel buffer via CMSampleBufferGetImageBuffer, append with presentation time
    5. Finish writing and return URL

    acceptance_criteria: File exists at Services/CircularFrameBuffer.swift, contains class CircularFrameBuffer, has append/flush/clear/bufferedDuration/isEmpty members.

    4. ADD method `configureHighFPSFormat(for device: AVCaptureDevice)`:
       - Enumerate `device.formats`
       - Filter formats where: (a) dimensions are at least 1280x720 but prefer closest to 720p (not 1080p+ to save bandwidth), (b) format supports video
       - From filtered formats, find the one with the highest `videoSupportedFrameRateRanges.maxFrameRate`
       - Prefer 240fps > 120fps > 60fps > 30fps
       - Call `device.lockForConfiguration()`, set `device.activeFormat` to chosen format, set `device.activeVideoMinFrameDuration` and `device.activeVideoMaxFrameDuration` to CMTime(value: 1, timescale: Int32(maxFPS))
       - Update `currentFPS` and `maxAvailableFPS` on main actor
       - If no high-FPS format found, leave default (30fps) and set currentFPS = 30

    5. REFACTOR `startRecording()`:
       - Create AVCaptureSession (do NOT set sessionPreset — activeFormat controls resolution)
       - Add video input from back camera
       - Create AVCaptureVideoDataOutput, set alwaysDiscardsLateVideoFrames = true, setSampleBufferDelegate to self on captureQueue
       - Add output to session
       - Call configureHighFPSFormat on the video device
       - Start session on background queue
       - Start timer for recordingDuration display (keep existing timer logic)
       - Set isRecording = true

    6. REFACTOR `stopRecording()`:
       - Keep existing timer invalidation
       - Stop session on background queue
       - Do NOT clear circular buffer here (buffer persists for challenge trigger)

    7. ADD method `saveBufferToDisk() async throws -> URL`:
       - Calls `circularBuffer.flush(to: tempURL, codec: .hevc, width: 1280, height: 720, fps: currentFPS)`
       - Sets `capturedVideoURL` to returned URL
       - Returns URL

    8. REFACTOR `cleanup()`:
       - Keep existing file cleanup
       - Also call `circularBuffer.clear()`

    9. ADD AVCaptureVideoDataOutputSampleBufferDelegate conformance (replace AVCaptureFileOutputRecordingDelegate):

       extension VideoCaptureManager: AVCaptureVideoDataOutputSampleBufferDelegate {
           nonisolated func captureOutput(_ output: AVCaptureOutput, didOutput sampleBuffer: CMSampleBuffer, from connection: AVCaptureConnection) {
               circularBuffer.append(sampleBuffer)
           }
       }
